状态机是一种用于管理系统生命周期中各种状态转换的**设计模式**。  

这里以订单为例来讲讲状态机。

## 什么是订单状态机
订单状态机是一种用于管理订单生命周期中各种状态转换的设计模式。

订单状态机定义了订单在整个生命周期中可能的状态，以及这些状态之间允许的转换规则。典型的订单状态包括：

+ 待付款 → 已付款 → 待发货 → 已发货 → 已签收 → 已完成
+ 还可能包括：已取消、退款中、已退款等状态



简单说就是集中一个类来管理订单的流转，流转的操作都调用状态机的方法进行。而之前要改变订单的状态都是分散在各个代码中的。

## 为什么需要状态机
**1. 业务规则约束**

确保状态转换符合业务逻辑（比如不能从"已发货"直接跳到"待付款"），防止非法的状态变更操作

**2. 数据一致性**

保证订单状态的完整性和正确性，避免并发操作导致的状态混乱（一般情况下会使用redis加入分布式锁防止并发对订单修改导致的错误）

**3. 可预测性**

明确定义每个状态下可以执行的操作，便于理解和维护业务流程

**4. 扩展性，便于管理**

新增状态或转换规则时，有清晰的框架可循，便于处理复杂的业务场景

## 示例代码
```plain
/**
 * 订单状态枚举
 */
public enum OrderStatus {
    PENDING_PAYMENT("待支付", "用户下单后的初始状态"),
    PENDING_SHIPMENT("待发货", "支付完成，商家准备发货"),
    SHIPPING("发货中", "商品在运输途中，用户可以追踪物流"),
    COMPLETED("已完成", "用户确认收货或自动确认后的最终状态"),
    CANCELLED("已取消", "任何环节都可能取消");

    private final String description;
    private final String detail;

    OrderStatus(String description, String detail) {
        this.description = description;
        this.detail = detail;
    }

    public String getDescription() {
        return description;
    }

    public String getDetail() {
        return detail;
    }

    /**
     * 获取可以转换到的状态
     */
    public static boolean canTransition(OrderStatus from, OrderStatus to) {
        if (from == null || to == null) {
            return false;
        }

        switch (from) {
            case PENDING_PAYMENT:
                return to == PENDING_SHIPMENT || to == CANCELLED;
            case PENDING_SHIPMENT:
                return to == SHIPPING || to == CANCELLED;
            case SHIPPING:
                return to == COMPLETED || to == CANCELLED;
            case COMPLETED:
            case CANCELLED:
                return false; // 终态，不能再转换
            default:
                return false;
        }
    }
}
```

```plain
/**
 * 订单状态机服务
 */
@Service
public class OrderStateMachineService {
    
    private static final Logger logger = LoggerFactory.getLogger(OrderStateMachineService.class);
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @Autowired
    private RedisLockUtil redisLockUtil;
    
    private static final String ORDER_KEY_PREFIX = "order:";
    private static final int LOCK_EXPIRE_TIME = 10000; // 锁过期时间10秒

    /**
     * 创建订单
     */
    public Order createOrder(String userId, String productName, BigDecimal amount) {
        String orderId = generateOrderId();
        Order order = new Order(orderId, userId, productName, amount);
        
        // 保存到Redis
        redisTemplate.opsForValue().set(ORDER_KEY_PREFIX + orderId, order);
        
        logger.info("创建订单成功: {}", order);
        return order;
    }

    /**
     * 支付订单
     */
    public boolean payOrder(String orderId) {
        return updateOrderStatus(orderId, OrderStatus.PENDING_SHIPMENT, "支付完成");
    }

    /**
     * 发货
     */
    public boolean shipOrder(String orderId) {
        return updateOrderStatus(orderId, OrderStatus.SHIPPING, "商品已发货");
    }

    /**
     * 确认收货
     */
    public boolean confirmOrder(String orderId) {
        return updateOrderStatus(orderId, OrderStatus.COMPLETED, "用户确认收货");
    }

    /**
     * 取消订单
     */
    public boolean cancelOrder(String orderId, String reason) {
        return updateOrderStatus(orderId, OrderStatus.CANCELLED, reason);
    }

    /**
     * 更新订单状态（线程安全）
     */
    private boolean updateOrderStatus(String orderId, OrderStatus targetStatus, String remark) {
        return redisLockUtil.executeWithLock(orderId, LOCK_EXPIRE_TIME, () -> {
            // 获取当前订单
            Order order = getOrder(orderId);
            if (order == null) {
                logger.warn("订单不存在: {}", orderId);
                return false;
            }

            // 检查状态转换是否合法
            if (!OrderStatus.canTransition(order.getStatus(), targetStatus)) {
                logger.warn("订单状态转换不合法: {} -> {}, orderId: {}", 
                    order.getStatus(), targetStatus, orderId);
                return false;
            }

            // 更新状态
            OrderStatus oldStatus = order.getStatus();
            order.setStatus(targetStatus);
            order.setRemark(remark);
            order.setUpdateTime(LocalDateTime.now());

            // 保存到Redis
            redisTemplate.opsForValue().set(ORDER_KEY_PREFIX + orderId, order);

            logger.info("订单状态更新成功: orderId={}, {} -> {}, remark={}", 
                orderId, oldStatus, targetStatus, remark);
            
            return true;
        });
    }

    /**
     * 获取订单信息
     */
    public Order getOrder(String orderId) {
        Object obj = redisTemplate.opsForValue().get(ORDER_KEY_PREFIX + orderId);
        return obj != null ? (Order) obj : null;
    }

    /**
     * 获取订单状态
     */
    public OrderStatus getOrderStatus(String orderId) {
        Order order = getOrder(orderId);
        return order != null ? order.getStatus() : null;
    }

    /**
     * 检查订单是否可以执行指定操作
     */
    public boolean canPerformAction(String orderId, OrderStatus targetStatus) {
        Order order = getOrder(orderId);
        if (order == null) {
            return false;
        }
        return OrderStatus.canTransition(order.getStatus(), targetStatus);
    }

    /**
     * 生成订单ID
     */
    private String generateOrderId() {
        return "ORDER_" + System.currentTimeMillis() + "_" + 
               UUID.randomUUID().toString().replace("-", "").substring(0, 8).toUpperCase();
    }
}
```

## 对账
通常订单还需要对账来检查状态一致性，因为如下原因：

+ 网络异常导致状态丢失
+ 服务重启导致状态不同步
+ 多系统状态不一致
