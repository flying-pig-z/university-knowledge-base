### 碎片清理
基于时间批量删除数据，然后要做碎片清理表的大小才会真正降低

### 批量删除
```plain
DELETE FROM table_hollis WHERE create_time < SUBDATE(CURDATE(),INTERVAL 3OO DAY);
```

这样的话当数据量大的时候会大范围加锁并且造成很大的磁盘IO压力。

可以像下面这样进行批量删除

1.获取要做数据清理的表的主键，或者非空唯一键的最大值和最小值。

如:

```plain
select min(id) as min_id,max(id) as max_id from table;
```

假如我们得到min_id = 100，max_id =100000;

2.分段取出第一个区间的所有数据，默认区间可能是1000，也可以根据binlog配置等进行调整。

```plain
SELECT id, (
    -- 子查询:检查这批数据的第一条是否需要删除
    SELECT 1 
    FROM (
        -- 取出这批数据的create_time
        SELECT create_time
        FROM table
        WHERE id >= 100 AND id <= 100000
        ORDER BY id ASC 
        LIMIT 1000
    ) t 
    WHERE create_time < SUBDATE(CURDATE(), INTERVAL 300 DAY) 
    LIMIT 1
) AS hasNeedDelItem
FROM table
WHERE id >= 100 AND id <= 100000
ORDER BY id ASC 
LIMIT 1000;
```

这段SQL代码的主要目的是查询出表table 中id值在100到100000之间的1000条记录，并为每条记录增加一个额外的字段hasNeedDelItem。这个字段用于标识是否存在一个条件满足的记录。
