+ 有分为离线全量对账和实时增量对账，分别适用于不同的数据量或实时要求
+ 数据量稍微大一点的对账一般不会在业务表直接进行，因为会影响业务，会在从库/大数据存储进行
+ 离线对账我比较熟悉，在多多的时候有个需求就是对账数仓平台和大数据开发平台的任务表和实例表，虽然不是我做的，但是大概流程就是将这两张表的数据拉到HIVE中，然后对相关字段计算MD5，然后JOIN得到不一致的数据
+ 实时增量对账一般接binlog同步到对账平台，调用相关引擎进行对账，可能设计Flink/Spark等实时计算引擎
+ 平账是指对账发现不一致的数据进行修复
