## 灰度
选取部分用户使用某个接口/功能。一般是随机选取，比如要选取百分之十的用户，可以根据userId取余10。

```plain
if ( userId % 10 == 0 ) {
  // 选取十分之一的用户
  doA();
} else {
  doB();
}
```

## AB测试
A/B测试作为数据驱动增长的核心工具，可以有效地提升流量的转化效率和产研的迭代效率.

<font style="color:rgb(89, 91, 102);background-color:rgb(250, 250, 250);">A/B 测试是一种</font><font style="color:rgb(54, 55, 64);background-color:rgb(250, 250, 250);">对比分析</font><font style="color:rgb(89, 91, 102);background-color:rgb(250, 250, 250);">方法，通过对流量进行细分和随机实验，并监控和跟踪实验效果，来判断实验所代表的策略的可行性和有效性。</font>

+ **<font style="color:rgb(54, 55, 64);background-color:rgb(250, 250, 250);">对比</font>**<font style="color:rgb(89, 91, 102);background-color:rgb(250, 250, 250);">：通过对流量的随机细分，来进行有效的独立随机实验，可以排除外在条件的影响，因此更加科学。</font>
+ **<font style="color:rgb(54, 55, 64);background-color:rgb(250, 250, 250);">分析</font>**<font style="color:rgb(89, 91, 102);background-color:rgb(250, 250, 250);">：通过跟踪和分析实验的事实效果数据，来判断实验的可行性和有效性，因此更加精确。</font>

![](https://cdn.nlark.com/yuque/0/2026/png/42768076/1767583429682-89bcff6f-29f5-4a25-a171-12ba855fd9be.png)

<font style="color:rgb(89, 91, 102);background-color:rgb(250, 250, 250);">相比原有基于时间的如T+30的效果对比，A/B测试可以排除时间和人为因素等外在所有因素的影响，并且保障同时进行的场景实验相互独立互不干扰，可以准确而且高效地评估实验效果。</font>

## 聚合写入
像比如点击量 点赞数这类的统计就可以在内存中先进行聚合，然后再写入，减少Mysql的写入量。
