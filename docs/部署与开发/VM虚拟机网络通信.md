【1】桥接模式

桥接模式使用的是物理网络，虚拟机直接接入物理接口。

比如连接手机热点，使用的是无线局域网适配器 WLAN，那么虚拟机也会接入WLAN，获取IP，网关和DNS。

桥接模式中虚拟机与物理主机在网络中效果差不多。

虚拟机和外网可以互相访问。

【2】NAT模式

NAT模式使用的是虚拟机提供的虚拟网络，默认接入的接口是VMware Network Adapter VMnet8。

使用NAT模式，虚拟机会提供虚拟DNS和DHCP服务，自动分配IP地址。

虚拟机通过网关(物理主机的虚拟网卡)直接访问物理主机。

如果要访问外部网络，则通过主机充当NAT，转化成物理网络，共享主机IP。

虚拟机可以访问外网，但外网无法直接访问虚拟机。

【3】仅主机模式

使用的是虚拟机提供的虚拟网络，默认接入的接口是VMware Network Adapter 仅主机模式VMnet1。

虚拟机会提供虚拟DNS和DHCP服务，自动分配IP地址

虚拟机通过网关(物理主机的虚拟网卡)直接访问物理主机。

不提供NAT功能，无法与外部网络通信，与外网完全隔离。



获取虚拟机的ip的方法：

（1）在虚拟机内ipconfig

（2）在宿主机中使用arp -a，找到对应接口对应主机
