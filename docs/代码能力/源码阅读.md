# 我的一些阅读源码的经验
+ 熟练掌握语法，语法很重要，不然有时候你根本不知道代码什么意思，为什么这样写就能实现功能
+ 使用gpt和copilot等工具辅助阅读代码来阅读。
+ 自顶向下。看源码，首先你要先会使用它。然后在使用过程中，从你感兴趣的Api/接口/功能/流程入手开始看，并分清主次，提取主要分支。由一个点慢慢扩散到其他点再慢慢扩散到面。
+ 先**结合官方文档和别人的博客和源码解析**对项目的结构和基本概念和项目的主要作用和主要流程有个整体的认知，并搞懂各个包的作用，快速定位到入手的关键类。从而更容易得阅读代码，快速定位到我们感兴趣的那部分类。
+ 多看代码，看多了代码你有时候看到类名或者包名或者方法名就能猜到这个东西是在干嘛？
+ 如果不是Spring项目，需要自己关注对象的生命周期。而且并不是单例，所以需要更关注代码的组织逻辑以及上下游的关系。’
+ 要学会对设计模式祛魅，设计模式只是代码的组织方式。该怎么实现还是怎么上实现，通过代码查看一层层的调用，我们就知道那个方法是怎么实现的。直接点点点就完事了。
+ 阅读代码很重要的两个字是**抽象**，要学会抽象概括，而不要沉迷于具体细节。
+ 边读边做笔记
+ 学会利用idea和github，比如在整个项目搜索词语 搜索类被哪个文件引用
+ 也会利用调试工具来进行堆栈分析
+ package-info.java是对整个包的注释 and 作用于整个包的注解，这个类可以看看
+ 看代码其实是推理，推理=线索+连接  
线索（项目文档 文章 文件注解 具体代码）  
连接（多看代码，对设计模式/语法/包名熟悉，一看就差不多懂了）  
另外还需要会利用工具（调试 搜索）
