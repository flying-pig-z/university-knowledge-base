### 并查集的应用场景与优势
1. 处理对象：Disjoint Set, 即 “不相交集合”。  
在一些应用问题中，需将 n 个[不同的](https://so.csdn.net/so/search?q=%E4%B8%8D%E5%90%8C%E7%9A%84&spm=1001.2101.3001.7020)元素划分成一组不相交的集合。开始时，每个元素自成一个单元素集合，然后按一定顺序将属于同一组元素的集合合并。其间要反复用到查询某个元素属于哪个集合的运算。适合于描述这类问题的抽象数据类型称为并查集。

![](https://i-blog.csdnimg.cn/blog_migrate/74ccb5f842fc2ca6f93717697ca81690.png)

2. 并查集（Union-Find Set）是一种高效解决  
（1）将两个元素所在的[集合合并](https://so.csdn.net/so/search?q=%E9%9B%86%E5%90%88%E5%90%88%E5%B9%B6&spm=1001.2101.3001.7020)  
（2）查询元素处在哪个集合的数据结构

### 一. 并查集结构
**并查集是一种树型结构**，这种树的要求比较简单：  
**1. 每个元素都唯一的对应一个结点；****  
****2. 每一组数据中的多个元素都在同一颗树中；****  
****3. 一个组中的数据对应的树和另外一个组中的数据对应的树之间没有任何联系；****  
****4. 元素在树中并没有子父级关系的硬性要求。**

**简单说就是同一组的元素可以看成是一棵树。**  
![](https://i-blog.csdnimg.cn/blog_migrate/d930c9defcf72e5b520af9eebb173250.png)

### 四. 查找算法的优化 -- 路径压缩
思考：上面查找的图，从 4 开始向上查找，当这颗树非常长的时候，是不是就要运算很多次？而且合并算法也要用到查找算法，所以当树 / 路径很长，就会导致查找算法和合并算法的复杂度很高。

**我们可以考虑在查找的同时将查找元素直接指向最顶的父节点。**

![](https://i-blog.csdnimg.cn/blog_migrate/7e70d7e900530acc41a0353e65b897e5.png)

**封装后的结构体（考试可以直接用）：**

```plain
#include<iostream>
#include<vector>
using namespace std;
struct League {
    vector<int> fa;
    int count=0;
    void init(int n) {
        fa.resize(n);
        for(int i=0;i<n;i++){
            fa[i]=i;
        }
        count=n;
    }
    int find(int i) {
        if(fa[i]==i)
            return i;
        else {
            fa[i]=find(fa[i]);
            return fa[i];
        }
    }
    void unionn(int i,int j) {
        int i_fa=find(i);
        int j_fa=find(j);
        if(i_fa!=j_fa) {
            fa[i_fa]=j_fa;
            count--;
        }
    }
};
```
