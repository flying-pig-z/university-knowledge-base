### 概述
#### 本质
回溯法也可以叫做回溯搜索法，它是一种搜索的方式。

回溯是递归的副产品，只要有递归就会有回溯。

回溯的本质是穷举，针对树状问题，比递推的栈模拟更符合思维。

#### 解决的问题
+ <font style="color:rgb(44, 62, 80);">组合问题：N个数里面按一定规则找出k个数的集合（组合无序，排列有序）</font>
+ <font style="color:rgb(44, 62, 80);">切割问题：一个字符串按一定规则有几种切割方式</font>
+ <font style="color:rgb(44, 62, 80);">子集问题：一个N个数的集合里有多少符合条件的子集</font>
+ <font style="color:rgb(44, 62, 80);">排列问题：N个数按一定规则全排列，有几种排列方式</font>
+ <font style="color:rgb(44, 62, 80);">棋盘问题：N皇后，解数独等等</font>

#### <font style="color:rgb(44, 62, 80);">决策树</font>
<font style="color:rgb(44, 62, 80);">回溯法解决的都是在集合中递归查找子集，集合的大小就构成了树的宽度，递归的深度就构成了树的深度。例如经典的组合问题：</font>

![](https://cdn.nlark.com/yuque/0/2025/png/42768076/1737462402067-a4a3faa2-b1e7-4e62-91e9-5a52a33d41d1.png)

<font style="color:rgb(44, 62, 80);">递归就要有终止条件，所以必然是一棵高度有限的树（N叉树）。</font>

#### <font style="color:rgb(44, 62, 80);">理解</font>
**回溯的每一层都代表着选择，通过递归进行下一次选择，通过删除路径撤销上一次的选择，通过终止条件代表到达了选择的尽头，将路径加入结果集代表保存选择。**

### <font style="color:rgb(44, 62, 80);">各个题型</font>
<font style="color:rgb(44, 62, 80);">组合问题：从n个数中选择k个数的组合，</font>**<font style="color:rgb(44, 62, 80);">不要求顺序</font>**<font style="color:rgb(44, 62, 80);">，但不能重复选择。</font>

+ <font style="color:rgb(44, 62, 80);">经典例题:子集生成、组合总和等</font>
+ **<font style="color:rgb(44, 62, 80);">关键是在递归时通过start参数控制选择范围,避免重复</font>**

<font style="color:rgb(44, 62, 80);">排列问题:对n个数进行全排列或部分排列，</font>**<font style="color:rgb(44, 62, 80);">考虑元素的顺序</font>**<font style="color:rgb(44, 62, 80);">，需要used数组标记已使用元素。</font>

+ <font style="color:rgb(44, 62, 80);">经典例题:全排列、字符串排列等</font>
+ **<font style="color:rgb(44, 62, 80);">重点是正确维护used数组的状态</font>**

> 1. 顺序重要性 
> + 组合问题：不考虑元素顺序，如[1,2]和[2,1]被视为同一组合
> + 排列问题：考虑元素顺序，[1,2]和[2,1]是两个不同的排列
> 2. 实现方式 
> + 组合问题：通常使用起始索引(start)参数来控制选择范围，避免重复
> + 排列问题：通常使用标记数组(used/visited)来记录哪些元素已被使用
> 3. 递归调用 
> + 组合问题：`backtrack(i+1)`，每次从当前元素之后开始选择
> + 排列问题：`backtrack()`，每次都从头开始选择，但跳过已使用元素
> 4. 结果数量 
> + 组合问题：从n个元素中选k个的组合数为C(n,k) = n!/(k!*(n-k)!)
> + 排列问题：从n个元素中选k个的排列数为P(n,k) = n!/(n-k)!
>

子集问题：生成一个集合的所有可能子集(幂集)，每个元素只有"选"与"不选"两种状态，不限制选取元素的个数(从0个到n个都考虑)。

+ 经典例题:子集、子集II(有重复元素)等
+ 关键是在每一步决策是否将当前元素加入子集，与组合问题相似，但在每个递归层级都收集结果，而非等到特定长度。【等到特定长度是直接返回上一层】

<font style="color:rgb(44, 62, 80);">字符串切割:将字符串切割成满足条件的子串，需要判断子串是否有效。</font>

+ <font style="color:rgb(44, 62, 80);">例如:分割回文串、IP地址划分等</font>
+ **<font style="color:rgb(44, 62, 80);">关键是定义好切割点的条件判断</font>**

<font style="color:rgb(44, 62, 80);">棋盘问题:在二维矩阵上放置特定元素，需要检查位置的合法性。</font>

+ <font style="color:rgb(44, 62, 80);">经典例题:N皇后、数独求解等</font>
+ **<font style="color:rgb(44, 62, 80);">重点是路径检查和状态重置</font>**

<font style="color:rgb(44, 62, 80);">图的遍历:寻找符合条件的路径，需要visited数组记录访问状态。</font>

+ <font style="color:rgb(44, 62, 80);">例如:迷宫问题、单词搜索等</font>
+ **<font style="color:rgb(44, 62, 80);">关键是维护好visited的状态</font>**

### 基本框架


<font style="color:rgb(44, 62, 80);">回溯三部曲：</font>

+ <font style="color:rgb(44, 62, 80);">回溯函数模板返回值以及参数</font>
    - <font style="color:rgb(44, 62, 80);">在回溯算法中，我的习惯是函数起名字为backtracking，这个起名大家随意。</font>
    - <font style="color:rgb(44, 62, 80);">回溯算法中函数返回值一般为void。</font>
    - <font style="color:rgb(44, 62, 80);">因为回溯算法需要的参数可不像二叉树递归的时候那么容易一次性确定下来，所以一般是先写逻辑，然后需要什么参数，就填什么参数。</font>
+ <font style="color:rgb(44, 62, 80);">回溯函数终止条件</font>
+ <font style="color:rgb(44, 62, 80);">回溯搜索的遍历过程</font>

> <font style="color:rgb(44, 62, 80);">不同题型有各自的特点,但基本框架是相似的:</font>
>
> <font style="color:rgb(44, 62, 80);">（1）定义结果集和路径</font>
>
> <font style="color:rgb(44, 62, 80);">（2）设置合适的终止条件</font>
>
> <font style="color:rgb(44, 62, 80);">（3）维护正确的状态</font>
>
> <font style="color:rgb(44, 62, 80);">（4）做出选择并递归</font>
>
> <font style="color:rgb(44, 62, 80);">（5）撤销选择并回溯</font>
>

**<font style="color:rgb(44, 62, 80);">最终模板</font>**

```java
void backtracking(参数) {
    if (终止条件) {
        存放结果;
        return;
    }

    for (选择：本层集合中元素（树中节点孩子的数量就是集合的大小）) {
        处理节点;
        backtracking(路径，选择列表); // 递归
        回溯，撤销处理结果
    }
}
```
