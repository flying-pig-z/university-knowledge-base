### Q：有路由器就不用交换机对吗
不对。

1. 路由器和交换机的定位不同

路由器主要完成网络间(例如公网和内网间)的转发,工作在网络层，交换机主要完成同一局域网内的数据转发,工作在数据链路层，虽然路由器内置了交换功能,但端口数量有限且性能较弱。

2. 端口数量需求

家用路由器通常只有4-8个LAN口，而一个局域网可能需要连接几十上百台设备，交换机可以提供24、48甚至更多端口。

3. 性能考虑

路由器内置的交换模块性能有限，独立的交换机有更强的交换容量和数据包处理能力，大型网络需要高性能交换机来保证数据传输效率。

4. 网络拓扑设计

典型的企业网络拓扑：

```plain
Internet -> 路由器 -> 核心交换机 -> 接入层交换机 -> 终端设备
```

这种分层设计可以:

提高网络可靠性，方便管理和扩展，实现更细致的流量控制。

所以在实际应用中,尤其是中大型网络,路由器和交换机都是必不可少的。它们各司其职,共同保证网络的正常运行。

### Q：交换机多次交换达到目的地，中间mac地址会改变吗
1. MAC 地址在交换机转发过程中的变化规则：

源 MAC 地址和目标 MAC 地址在整个二层网络(同一局域网)中保持不变，只有跨网段(经过路由器)时，MAC 地址才会改变。

2. 举个例子：

```plain
PC1(192.168.1.100) -> 交换机A -> 交换机B -> 交换机C -> PC2(192.168.1.200)
MAC地址保持不变：
[源MAC: PC1的MAC] [目标MAC: PC2的MAC]
```

但如果跨网段：

```plain
PC1(192.168.1.100) -> 交换机A -> 路由器 -> 交换机B -> PC2(192.168.2.200)
                       [PC1 MAC -> 路由器MAC]  [路由器MAC -> PC2 MAC]
```

3. 完整的转发流程：

交换机A收到帧，学习源MAC地址，查MAC地址表，找到下一个转发端口，不改变MAC地址，直接转发，交换机B、C重复相同过程，最终帧到达目标主机。

**总结：**

**同一局域网内，MAC地址保持不变，只有经过路由器才会改变MAC地址，交换机通过MAC地址表进行转发决策。**
