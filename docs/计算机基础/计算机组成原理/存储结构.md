## 存储器层次
### 层次结构
存储器层次结构按照速度、容量和成本从高到低组织不同类型的存储器，充分利用了程序的局部性原理。

主要分为以下几个层次：

1. 寄存器 -- Register

位于CPU内部，速度最快（访问时间约为1ns），容量最小（几百字节到几KB），成本最高，用于存储CPU当前正在处理的数据和指令

2. 高速缓存 -- Cache

分为L1、L2、L3级缓存，L1 Cache直接集成在CPU核心中，速度极快（约3-4ns），L2、L3 Cache容量更大但速度稍慢（L2约10ns，L3约20-30ns）

用于缓存频繁使用的数据，减少访问主存的次数

3. 主存储器 -- Main Memory

通常指RAM（随机存取存储器），访问速度中等（约50-100ns），容量较大（现代个人电脑通常8GB-32GB或更多），直接与CPU交换数据，掉电数据丢失

4. 固态硬盘 -- SSD/ 机械硬盘 -- HDD

作为外部存储设备，访问速度相对较慢（SSD约100μs，HDD约10ms），容量大（从几百GB到几TB），成本相对较低，掉电数据不丢失



存储器层次结构的工作原理：

+ 上层存储器作为下层存储器的高速缓冲
+ 每一层存储器都存放着下一层存储器中最活跃的数据副本
+ 随着层次升高，存储器的速度越快、容量越小、每位价格越高

这种层次化的设计能够很好地平衡性能、容量和成本这三个关键因素，使得计算机系统能够高效地运行。通过合理利用程序的时间局部性和空间局部性，大大提高了整个存储系统的性能。

### 层次关系
访问速度逐层递减，容量逐层递增，单位成本逐层递减，带宽逐层递减。

寄存器直接集成在CPU内部，Cache通常也集成在CPU芯片内（L1、L2通常在CPU核心内，L3是多核共享），主存通过内存总线与CPU连接，外部存储通过I/O总线与系统相连。

## CPU缓存一致性
###  写入策略  
**写直达(Write Through)**

1. 工作流程:

当 CPU 写入数据时，同时写入 Cache 和主存，Cache 和主存的数据始终保持一致

2. 优点:

数据一致性好,主存总是包含最新数据；系统崩溃时数据丢失风险低；实现简单,易于管理

3. 缺点:

每次写操作都要访问主存,性能较差；产生大量内存总线流量；写入延迟较高

**写回(Write Back)**

1. 工作流程:

CPU 写入数据时只写入 Cache；使用脏位(Dirty Bit)标记被修改的 Cache Line；当 Cache Line 被替换或主动刷新时,才写回主存

2. 优点

减少内存访问次数,性能更好；降低总线带宽占用；特别适合频繁写入的场景

3. 缺点

实现复杂,需要额外的脏位管理；系统崩溃时可能丢失尚未写回的数据；Cache 一致性协议更复杂



**实际应用**

1. 现代处理器通常采用写回策略:

性能要求高，有完善的断电保护机制，配合 MESI 等缓存一致性协议使用。

2. 特殊场景仍会使用写直达:

实时系统对数据一致性要求高，嵌入式设备没有备用电源，简单的单处理器系统。

这两种策略各有优劣,选择哪种取决于具体应用场景的需求。现代处理器通常会采用写回策略,并通过各种优化手段来提升性能。

### 读写流程
**读取数据**

（1）CPU 首先检查所需数据是否在寄存器中

（2）如果不在寄存器,则依次查找各级缓存

（3）如果缓存也未命中,则从内存读取数据

（4）数据通过系统总线传输到 CPU

（5）CPU 将数据加载到寄存器中进行处理

**写入数据**

（1）CPU 首先将数据写入寄存器

（2）根据缓存策略(如写直达、写回等)更新缓存

（3）必要时将数据写回内存

（4）数据通过系统总线传输

（5）最终写入目标存储位置



### 缓存一致性问题
缓存一致性问题产生于多处理器(或多核)系统中,当多个处理器都有自己的私有缓存时,可能会导致同一内存地址的数据在不同缓存中有不同的值。

**具体场景举例**  
假设有两个 CPU 核心(CPU1 和 CPU2)共享一块内存:

1. 初始状态:

内存地址 A 中的值为 0，CPU1 和 CPU2 的缓存都从内存读取了这个值。

2. 问题产生:

```plain
CPU1: 将 A 修改为 1 (写入自己的缓存)
CPU2: 读取 A 的值 (从自己的缓存读取,得到旧值 0)
```

3. 结果:

CPU1 的缓存中 A = 1，CPU2 的缓存中 A = 0 ，内存中的值取决于写入策略。

这就产生了数据不一致的问题。

**导致的问题**

1. 数据错误：程序可能读取到过期的数据，不同处理器看到的数据状态不一致。
2. 并发问题：影响同步操作的正确性，可能破坏锁机制的有效性。
3. 性能下降：需要额外的同步开销，可能导致缓存失效。

**解决方案**

1. 硬件层面:
+ 缓存一致性协议(如 MESI)
+ 总线嗅探机制
+ 目录协议
2. 软件层面:
+ 内存屏障指令
+ volatile 关键字
+ 原子操作

### 总线嗅探
总线嗅探(Bus Snooping)是一种维护缓存一致性的硬件机制,所有处理器都会不断监听(嗅探)系统总线上的数据传输活动。

**工作原理**

1. 监听机制

每个处理器的缓存控制器都有一个嗅探器(snooper)，不断监听总线上的数据传输和地址信息，特别关注与自己缓存中数据相关的传输。

2. 典型场景

```plain
CPU1 要修改数据 X:
- 在总线上广播"我要修改X"的消息
- CPU2 发现自己缓存里有 X
- CPU2 将自己的 X 副本标记为无效
- CPU1 获得独占权,可以修改 X
```

3. 状态变化

```plain
当监听到相关操作时:
Read: 其他处理器读取数据
→ 如果本地是 Modified,需要写回
→ 状态可能变为 Shared

Write: 其他处理器写入数据
→ 如果本地有副本,标记为 Invalid
→ 确保只有一个处理器能修改数据
```

**优缺点**

优点:

实现简单直观，响应速度快，硬件自动处理,对软件透明。

缺点:

总线带宽消耗大，处理器数量增加时扩展性差，系统总线可能成为瓶颈。

**应用场景**

1. 小规模系统:

2-4 个处理器的系统，共享总线架构，处理器之间通信频繁。

2. 替代方案:

大规模系统使用目录协议，分布式系统使用一致性协议，NUMA 架构使用特殊的互联机制。

总线嗅探是一种重要的缓存一致性维护机制,在小规模多处理器系统中应用广泛。理解它的工作原理对于深入理解计算机体系结构很有帮助。

### MESI 协议
MESI 协议是最常用的缓存一致性协议,将缓存行标记为四种状态，这四种状态分别对应其四个字母。

1. Modified(修改态)

该缓存行已被修改,数据与内存不一致，只有这个 CPU 拥有此数据的副本，其他 CPU 必须等待数据写回后才能读取。

2. Exclusive(独占态)

数据未被修改,与内存一致，只有这个 CPU 拥有此数据的副本，可以直接修改进入 Modified 状态。

3. Shared(共享态)

数据未被修改,与内存一致，多个 CPU 都有这份数据的副本，需要先让其他副本无效才能修改。

4. Invalid(无效态)

该缓存行无效，必须从内存或其他 CPU 缓存重新读取，表示数据已过期。

**状态转换**

1. 读请求触发的转换:

```plain
当CPU发起读请求:
Invalid → Shared(其他CPU有副本)
Invalid → Exclusive(没有其他CPU有副本)
```

2. 写请求触发的转换:

```plain
当CPU发起写请求:
Modified → Modified(本地写入)
Exclusive → Modified(本地写入)
Shared → Modified(需要先广播使其他副本失效)
Invalid → Modified(需要先获取最新数据)
```

3. 其他 CPU 操作触发的转换:

```plain
当其他CPU读取:
Modified → Shared(需要写回)
Exclusive → Shared
```

**具体例子**

```plain
初始状态: CPU1和CPU2都没有数据X的副本(Invalid)

1. CPU1读取X:
- CPU1状态: Invalid → Exclusive
- CPU2状态: 保持 Invalid

2. CPU2也来读X:
- CPU1状态: Exclusive → Shared
- CPU2状态: Invalid → Shared

3. CPU1要修改X:
- CPU1状态: Shared → Modified
- CPU2状态: Shared → Invalid
```

**优缺点**

优点:

有效确保缓存一致性，支持写回策略,提高性能，状态转换清晰,易于实现。

缺点:

需要额外的状态存储开销，状态转换复杂，总线流量较大。

MESI 协议是现代处理器中最常用的缓存一致性协议之一,理解它对于深入理解多核系统的内存一致性很有帮助。

## 伪共享问题
CPU 缓存是以缓存行(Cache Line)为基本单位,通常是 64 字节，当多个线程访问同一缓存行中的不同变量时,就会发生伪共享，即使线程操作的是不同的变量,但因为它们在同一缓存行,也会导致缓存一致性开销。
