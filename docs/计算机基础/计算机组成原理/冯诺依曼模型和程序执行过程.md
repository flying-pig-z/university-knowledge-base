### 冯诺依曼模型的基本组成
1. 存储器：存储程序和数据
2. 控制器：负责指令的解码和控制各部件工作
3. 运算器：进行算术逻辑运算
4. 输入设备：接收外部输入
5. 输出设备：输出计算结果

### 程序执行过程（指令周期）
1. **取指令（Fetch）**
+ CPU从程序计数器(PC)指示的地址取出指令
+ 指令被加载到指令寄存器(IR)
+ PC自动递增，指向下一条指令
2. **指令译码（Decode）**
+ 控制器解析IR中的指令
+ 确定指令类型（如加法、移动等）
+ 识别操作数地址
3. **执行指令（Execute）**
+ 根据指令类型执行相应操作：
    - 算术运算：在ALU中进行计算
    - 数据移动：在寄存器/内存间传送数据
    - 控制转移：修改PC的值
+ 结果存入指定位置

### 例1：一条简单的加法指令执行过程:
```plain
ADD R1, R2, R3  ; R1 = R2 + R3

1. 取指：从内存读取ADD指令到IR
2. 译码：解析出是加法操作，需要用到R2和R3
3. 执行：
   - 从R2、R3读取操作数
   - ALU执行加法运算
   - 结果写入R1
```

这个周期会不断重复，直到程序执行完毕。现代计算机通过流水线等技术优化了这个过程，但基本原理仍然遵循冯诺依曼模型。

### 例2：a = 1 + 2执行过程
1. **取指令阶段**

从内存加载操作数1和2，从内存加载加法指令。

2. **译码阶段**

控制器解析出这是加法运算，确定操作数位置和存储位置。

3. **执行阶段**

ALU执行1+2的加法运算，计算结果3临时存入寄存器。

4. **存储结果**

分配内存空间给变量a，将结果3从寄存器存入a的内存地址。

机器码层面：

```plain
LOAD R1, #1     ; 将1加载到寄存器R1
LOAD R2, #2     ; 将2加载到寄存器R2
ADD R3, R1, R2  ; R1和R2相加，结果存入R3
STORE R3, a     ; 将结果存入变量a的内存位置
```
