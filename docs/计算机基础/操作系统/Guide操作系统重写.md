# 操作系统基础
## 什么是操作系统？
计算机硬件系统上配置的第一个大型软件，称为计算机操作系统，如果该软件满足：

(1)控制计算机系统的工作流程；   
(2)管理计算机系统的硬件和软件；   
(3)为其他软件和用户提供安全、方便的运行、操作环境；  
(4)提高计算机系统的效率。

## 用户态和内核态
### 什么是用户态和内核态
处理器能处理的指令分为：

+ 特权指令：需要特殊权限才可使用的指令，通常涉及系统安全
+ 非特权指令：任何情况下都可使用的指令

处理器的控制寄存器中用专门的位（程序状态字）标识当前的工作模式：

+ 核心态（管态）：已获得特殊权限的状态，能够执行所有指令
+ 用户态（目态）：只能执行非特权指令

## 系统调用
### 什么是系统调用
一组由操作系统设计者编写的内核中的子程序，用以封装用户（程序员）可能使用到的操作系统功能。

系统调用所涉及的子程序属于操作系统内核的一部分，保存于内核空间中，应用程序不能直接访问，于是需要通过特殊方式才能够实现系统调用。

### 系统调用的过程了解吗？
系统调用的实现：   
1. 放置调用号标识   
2. 访管指令：发出表示系统调用请求的中断（软中断、陷阱）   
3. 处理器保护程序现场，转向中断处理（内核态）   
4. 中断处理程序识别中断为系统调用请求，于是转向内核的访管指令处理程序   
5. 访管指令处理程序在地址入口表中查找所请求的调用号对应的内核子程序的入口地址   
6. 从查找到的地址开始执行内核子程序（系统调用）并得到结果   
7. 回复应用程序现场，返回用户态

# 进程和线程
## 什么是进程和线程？
进程：进程就是一道程序在一个数据集的执行过程。

线程：线程是比进程更小的单位。把进程细化成若干个可以独立运行的实体，每一个实体称为一个线程。

## 有了进程为什么还需要线程?
+ 线程更加轻量级
+ 可以减少切换开销
+ 同一进程内的线程共享内存和文件，因此它们之间相互通信无须调用内核。
+ 多个线程可以并发处理不同的任务，更有效地利用了多处理器和多核计算机。而进程只能在一个时间干一件事，如果在执行过程中遇到阻塞问题比如 IO 阻塞就会挂起直到结果返回。

## 为什么要使用多线程?
充分的利用系统资源，提高任务的执行效率。

## 线程间的同步的方式有哪些？
锁和信号量

> 锁一般用于资源互斥，而信号量更加灵活，一般用于控制资源数量。

## [<font style="color:rgb(60, 60, 67);">PCB 是什么？包含哪些信息？</font>](https://javaguide.cn/cs-basics/operating-system/operating-system-basic-questions-01.html#pcb-%E6%98%AF%E4%BB%80%E4%B9%88-%E5%8C%85%E5%90%AB%E5%93%AA%E4%BA%9B%E4%BF%A1%E6%81%AF)
进程控制块（PCB）是操作系统用以描述进程信息的数据结构，由三部分组成：

**基本信息部分**

进程名：通常是程序名进程  
标识符(pid)：操作系统用以识别进程的唯一标识  
用户标识：创建进程的用户标识  
进程状态：运行、就绪或阻塞

**管理信息部分**

程序和数据地址：程序和数据在内存中的位置信息  
IO操作相关参数：进程进行IO操作时的参数  
进程通信信息：用于进程间通信的消息队列指针等

**控制信息部分**

现场信息：进程离开运行状态时，保存进程现场以备恢复  
调度参数：与进程调度相关的参考信息，如优先级、运行时间等  
同步、互斥信息量：用户保证进程间同步、互斥关系的信息

> <font style="color:rgb(60, 60, 67);">当操作系统创建一个新进程时，会为该进程分配一个唯一的进程 ID，并且为该进程创建一个对应的进程控制块。当进程执行时，PCB 中的信息会不断变化，操作系统会根据这些信息来管理和调度进程。</font>
>

## 进程有哪几种状态?
进程有三种状态，就绪 运行 阻塞，如果算上前后的创建和结束状态，那就有五种状态，分别如下：

+ **创建**：进程正在被创建，尚未到就绪状态。
+ **就绪**：能够运行但却未在被处理器处理的状态
+ **运行**：正在被处理器处理的状态
+ **阻塞**：由于各种原因（IO操作、资源缺乏等）暂时无法运行的状态
+ **结束**：进程正在从系统中消失。可能是进程正常结束或其他原因中断退出运行。

## 什么是僵尸进程和孤儿进程？
在 Unix/Linux 系统中，子进程通常是通过 fork()系统调用创建的，该调用会创建一个新的进程，该进程是原有进程的一个副本。子进程和父进程的运行是相互独立的，它们各自拥有自己的 PCB，即使父进程结束了，子进程仍然可以继续运行。

当一个进程调用 exit()系统调用结束自己的生命时，内核会释放该进程的所有资源，包括打开的文件、占用的内存等，但是该进程对应的 PCB 依然存在于系统中。这些信息只有在父进程调用 wait()或 waitpid()系统调用时才会被释放，以便让父进程得到子进程的状态信息。

这样的设计可以让父进程在子进程结束时得到子进程的状态信息，并且可以防止出现“僵尸进程”（即子进程结束后 PCB 仍然存在但父进程无法得到状态信息的情况）。

+ **僵尸进程**：子进程已经终止，但是其父进程仍在运行，且父进程没有调用 wait()或 waitpid()等系统调用来获取子进程的状态信息，释放子进程占用的资源，导致子进程的 PCB 依然存在于系统中，但无法被进一步使用。这种情况下，子进程被称为“僵尸进程”。避免僵尸进程的产生，父进程需要及时调用 wait()或 waitpid()系统调用来回收子进程。
+ **孤儿进程**：一个进程的父进程已经终止或者不存在，但是该进程仍在运行。这种情况下，该进程就是孤儿进程。孤儿进程通常是由于父进程意外终止或未及时调用 wait()或 waitpid()等系统调用来回收子进程导致的。为了避免孤儿进程占用系统资源，操作系统会将孤儿进程的父进程设置为 init 进程（进程号为 1），由 init 进程来回收孤儿进程的资源。

## 如何查看是否有僵尸进程？
Linux 下可以使用 Top 命令查找，zombie 值表示僵尸进程的数量，为 0 则代表没有僵尸进程。

![](https://cdn.nlark.com/yuque/0/2024/jpeg/42768076/1734662904237-5b939307-208b-4ff2-93d4-287a8d03697f.jpeg)

# 死锁
## 什么是死锁？
一组进程均处于阻塞状态，其中的任一进程阻塞状态的解除都依赖于另一进程的后续操作，这种系统状态称作死锁（Deadlock）。

![](https://cdn.nlark.com/yuque/0/2024/png/42768076/1734583825104-971fb88a-43a4-4cfe-9e52-67c99c90a669.png)

## 产生死锁的根本原因和四个必要条件是什么？
死锁的根本原因是可用资源的数量小于所需资源的数量。

死锁发生的必要条件：

+ 互斥条件——请求的资源要是互斥的，不能被共享
+ 不剥夺条件——当进程申请资源时不能剥夺其他进程正在使用的资源
+ 请求与保持（部分分配）条件——进程在申请资源不成功时仍然保持已申请的资源
+ 环路等待条件——进程相互等待关系形成环路

> 你就想象五个哲学家五个刀叉吃饭那个场景进行记忆。
>
> 首先，刀叉只能一个人拿，然后，你不能把别人刀叉抢过来，然后，你手里拿着一个刀叉，还去请求拿别人的刀叉，最后，你们的请求要形成一个环。
>

## 解决死锁的方法
### 死锁的预防 -- 资源分配时破坏4哥必要条件
**通过在资源分配时采取一系列限制措施，破坏4条必要条件之一，以预防死锁的发生。 **

1. 互斥条件的破坏 

+ 方法：不允许申请临界资源，临界资源需要转化为非临界资源方可被申请和使用。   
缺陷：大部分情况下无法实现，有很多资源往往是不能同时访问的。 

2. 不剥夺条件的破坏 

+ 方法：允许进程申请资源时剥夺其他进程正在使用的资源。   
缺陷：被剥夺资源的进程不可预期地被打断；系统需处理“回退”；缺乏公平性；反复剥夺与被剥夺。

3. 请求与保持条件的破坏 

+ 方法1：资源暂时归还——进程申请资源不成功时，暂时归还已分配的资源。【相当于数据库检测到发生死锁超时就rollback回滚】   
缺陷：系统需处理“回退”；反复申请与归还。 
+ 方法2：静态分配——进程需要一次申请完所有可能使用的资源。 【就比如数据库的意向锁】  
缺陷：难以预知需要使用的资源；降低资源利用率。

4.环路等待条件的破坏

+ 方法1：单请求方式——资源申请不允许嵌套（但可一次申请多个），归还旧资源后方可申请新资源。  
【和上面的方法2差不多，都是要一次性申请完归还后才能再申请。也是像数据库的意向锁】  
缺陷：同时需要多个资源时只能实行小范围内的“静态分配”。
+ 方法2：按序分配——对资源编号，进程只允许申请比当前已拥有资源编号更大的资源。  
缺陷：若使用大编号资源的段内需使用小编号资源，则只能实行小范围内的“静态分配”；编号管理困难。

> 一般实用的预防死锁的方法，是后两种，即通过考虑破坏请求与保持条件和环路等待条件。
>

### 死锁的避免 -- 资源分配时进行检查
死锁的避免：进程可以自由地动态地申请资源，但分配资源时对可能发生的死锁进行预测，以决定是否立即分配。

最常使用的就是银行家算法。银行家算法是怎么样的呢？

首先要先来介绍一下安全状态。安全状态就是可以找到一个进程执行的顺序，使得无论这些进程内部此后如何申请和归还资源，总能按此顺序完成这组进程。这个执行顺序称作安全序列。

我们一般在最坏情况下模拟运行来判断某个状态是否是安全状态。（结合所需各个进程所需的最大资源和各个进程已经使用的资源来模拟运行）

而银行家算法就是在每次资源申请之前都假设已经分配使用，然后模拟运行来判断系统是否还处于安全状态，如果是就允许分配，否则就阻止分配。

### 死锁的检测 -- 资源分配后定期检查
若系统未采取任何预防死锁或避免死锁的措施，则可以安排专门的检测程序，定期对系统进行检查，以发现正在发生的死锁，并将其恢复。

死锁检测的思路：“最好情况”下的模拟运行

+ 假设各进程在满足了当前的资源申请后将立即归还已分配资源
+ 试图找到满足所有进程当前资源申请的分配顺序

我们利用**资源分配图**的化简来表达“模拟运行”过程，从而检测死锁。具体方法见JavaGuide。

如果类比悲观锁和乐观锁，死锁的避免就是悲观锁，假设最坏的情况；死锁的检测就像是乐观锁，假设最好的情况。

### 死锁的恢复/解除 -- 死锁发生后进行解除
死锁的恢复主要有以下三种方法：

+ 剥夺资源：在死锁发生时（与预防阶段不同），剥夺（某个）死锁进程的资源.
+ 撤销进程：在死锁发生时撤销（某个）死锁的进程。
+ 重新启动系统：在死锁发生时重新执行所有进程。

鸵鸟算法：由于死锁现象是小概率事件，现有的操作系统一般不考虑它的存在，而不做特殊应对。

# 内存管理
## 什么是内存碎片
内存碎片就是内存中那些很难或者无法被程序使用的区域。

内存碎片主要分为内碎片和外碎片。

内碎片就是说在单一分区管理或者固定分区管理的时候，会分配给程序一个固定大小的空间，一个空间就只能放一个程序。然后分配的空间的话很有可能超过程序的大小，所以肯定有一小部分空间是剩下来而且由于一个空间只能放一个程序，所以没办法被其他程序利用的。就形成了内存碎片。

![](https://cdn.nlark.com/yuque/0/2024/png/42768076/1734408987498-56562f63-9240-4fba-9197-31772a9cab2b.png)![](https://cdn.nlark.com/yuque/0/2024/png/42768076/1734408980516-cb221ada-cd9d-4ba5-85de-7b6f302e9fb7.png)

外碎片又是什么呢？

操作系统会在上面分区管理的基础上进行改进成可变分区的管理，就是说给每个程序分配对应大小的空间，而不是像上面那样划分成一个个的区域再分配，这样就不会产生内碎片的问题。但是的话系统分配的时候，比如说A和C之间有一块内存空间，这时候载入B，但是B的话没有占满整个空间，这时候可能剩下一个很小的内存区域，这个区域也能载入程序，但是需要所需内存比它小的程序才能载入。这就造成了这块区域很难被利用，所以就形成了内存碎片，称为外碎片。

![](https://cdn.nlark.com/yuque/0/2024/png/42768076/1734408905040-cff965c6-d2c6-40cc-a1fd-fc2cc938fd5d.png)

如果内存碎片过多的话就会造成内存的利用率下降。

![](https://cdn.nlark.com/yuque/0/2024/png/42768076/1734409956533-743aed25-ad99-47f2-a67e-543de2ecdf45.png)

## 分页机制
简单说就是先对内存和程序进行分页（分块），分成等长的一页一页等长的内存区域。（这些区域的大小一般是2的k次方字节）。

然后程序一页一页对照着装入内存。

注：程序是连续装入的，但是程序的页在内存中不一定是连续的。

### 页表有什么用？地址翻译过程是怎么样的？
页表用来记录进程中的页号与内存中的页号的对应关系。（也可以看做是虚拟地址和物理地址的映射）

在分页机制下，每个应用程序都有一个对应的页表。

那么虚拟地址怎么变为物理地址呢？

分页机制下的虚拟地址由两部分组成：

+ 页号：通过虚拟页号可以从页表中取出对应的物理页号；
+ 页内偏移量：物理页起始地址+页内偏移量=物理内存地址。

具体的地址翻译过程如下：

1.MMU 首先解析得到虚拟地址中的虚拟页号；  
2.通过虚拟页号去该应用程序的页表中取出对应的物理页号（找到对应的页表项）;  
3.用该物理页号对应的物理页起始地址（物理地址）加上虚拟地址中的页内偏移量得到最终的物理地址。

![](https://cdn.nlark.com/yuque/0/2024/png/42768076/1734708239663-98062b89-e1c3-45c1-9162-ed8623340870.png)

### 单级页表有什么问题？为什么需要多级页表？
页面太多，页表太大，查询变慢。多级页表空间换时间，减少查询次数增快查询的速度。

1. 先进先出（FIFO）算法 -- 选择存在于内存中最久的页作为淘汰页

2. 最久未使用（LRU）算法 -- 选择最长时间未被使用的页作为淘汰页

3.最近未使用（NRU）算法 -- LRU的近似实现

定期重置A=0，选择最近（上次重置以来）未被使用的某页

4.最不常用（LFU）算法 

记录页面使用次数，选择最少被使用的页作为淘汰页

5.二次机会（Second Chance）算法

在FIFO的基础上选择淘汰页，但淘汰前进行检查，若选择到的页A=1，则不淘汰此页，重置其A=0并将其放置回页序列末端，并转而选择下一页并检查。

## 局部性原理
程序的局部性原理：

+ 在程序运行过程的一个较小时间范围内，只需要一小部分的程序信息，其他部分暂时不需要。
+ 在程序的一次执行过程中，程序的所有指令和数据被访问的概率间存在不平衡性。

程序的局部性的规律：

+ 时间局部性：某地址的信息被访问后，短时间内很可能再次被访问。

因为时间的局部性，所以分页机制会将最近访问的页放入缓存中，如果下一次访问的页也在缓存中，就不需要再次访问内存，而是直接从缓存中读取。

+ 空间局部性：某地址的信息被访问后，与其临近的信息很可能被访问。

因为空间局部性，分页机制中通常采用预取技术来预先将相邻的一些页读入内存缓存中，以便在未来访问时能够直接使用，从而提高访问速度。
