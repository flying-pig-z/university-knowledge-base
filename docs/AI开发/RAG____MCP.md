## RAG -- 检索增强
以个人知识库 + AI为例。



先将个人知识库的文章去调用embding api计算出对应的向量，将文章和向量关联存储到数据库中。

> 如果是二维向量就是 123,345 类似这种形式，如果是三维向量就是123,345,567 类似这种形式，以此类推
>

这里为了防止文章过长，我们一般会将文章进行切割，这里可以按照语意分割，也还是简单的固定大小比如500个一块的切割，分成不同大小的文字块，然后调用embding api计算出向量，将这块文字和向量关联存储到数据库中，而不是整篇文章。

然后用户输入问题过后，将问题也转化为向量，然后去数据库中知识库中的向量计算余弦相似度，将相似度高的向量对应的文字块筛选出来，然后和问题组合一起喂给AI，AI返回结果。

> 如果数据量大一般使用向量数据库加速检索，因为如果是mysql本质上就是暴力遍历，要将所有文章/文字块的向量和问题的向量计算余弦相似度
>
> 而向量数据库这块则不需要，它使用ANN（近似最近邻）算法加速检索，最常用的是 HNSW（分层可导航小世界图） 
>
> 就比如去图书馆找书，Mysql需要一个个找并计算，而图书馆已经按"相似性"分区，先去大致区域，再细找，从而加速查找  
>

这样就实现了AI可以实现通过用户的问题检索到知识库中对应的知识进行回答。



## MCP
MCP本质上是一种协议。

MCP客户端相对较少，最为出名就是Claude Destop。

想Spring Cloud Alibaba AI就直接使用@McpTool注解标注方法就可以开放MCP接口，然后你在Claude Destop上配置MCP服务的地址就可以开启MCP功能。

这时候Claude Destop就知道你有哪些MCP接口，并且在执行问题/任务的时候自动决策需不需要调用接口，调用哪个接口，并且询问你要不要执行调用。
