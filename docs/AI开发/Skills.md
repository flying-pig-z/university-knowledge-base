### 一、Prompt 提示词
要让 AI 干活，最基本的方式是发一条指令（prompt）。为了让效果更精准，可以加入格式、步骤等规则，形成**结构化提示词**——规定得越细越好，但手敲太长不现实。

---

### 二、Command 命令
将常用的长提示词**固化成文件**，通过短命令快速调出。本质是"短命令 → 替换为一段固定内容"的快捷方式。Claude Code 等工具都支持此功能。

---

### 三、System Prompt 系统提示词
用户在输入框敲的内容默认是 **User Prompt（用户提示词）**，还有一层优先级更高的 **System Prompt（系统提示词）**，大模型对系统提示词的指令遵循效果更好。

一些 AI 软件允许用文件形式存放系统提示词，在后台自动注入，如 Cursor 的 `.cursorrules`、Claude Code 的 `claude.md`。

---

### 四、Metadata 元数据
面对多场景需求（写简历/周报/邮件），将规则拆成多个 MD 文件，在每个文件头部加一段**简短描述**，说明该文件的用途和触发条件——这就是 **Metadata（元数据）**。

工作流程：

1. AI 客户端只将各文件的 metadata（很小，省 token）发给大模型
2. 大模型判断场景，返回需要哪个文件
3. 客户端加载对应文件的完整内容作为系统提示词，再发给大模型

---

### 五、References 与 Scripts
随需求细化，单个文件也会变大，进一步拆分：

+ **References（参考资料文件夹）**：大模型按需读取，用到哪份才加载哪份，即"渐进式披露"，大量节省 token。
+ **Scripts（脚本文件夹）**：大模型可执行本地代码脚本，实现纯文本对话之外的能力（如将内容写入 Word 并导出 PDF）。

---

### 六、Skill 是什么
将以上所有部分打包成一个文件夹：

```plain
resume-writer/         ← Skill 名称
├── skill.md           ← 主入口（原系统提示词）+ metadata
├── references/        ← 参考资料，按需加载
└── scripts/           ← 可执行脚本
```

这个**以文件夹形式存在、可动态加载的系统提示词集合**，就是 **Skill**。

安装即放入 `skills` 目录，AI 客户端自动识别。

---

### 七、Skills vs MCP vs Workflow
| 概念 | 本质 | 比喻 |
| --- | --- | --- |
| **MCP** | 让大模型操控外部工具的协议 | 给大脑配的"手"，工具是手上的器械 |
| **Skills** | 规定何时、按何顺序使用工具的操作经验 | 告诉手"怎么用"工具的经验手册 |
| **Workflow** | 通过规则配置编排多步骤流程（如 N8N） | 固定的流水线，结构在设计阶段就定死 |


Skills 可简单理解为**大模型驱动的 Workflow**：Workflow 流程结构固定，Skills 由大模型动态决策，灵活性更高，但两者最终能实现相似功能。
