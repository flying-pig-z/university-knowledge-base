### List
1. Vector - 最早的线程安全List实现，所有方法都是同步的，但效率较低
2. Collections.synchronizedList() - 通过装饰器模式将普通List转换为线程安全的版本
3. CopyOnWriteArrayList - 适用于读多写少的场景，读操作不加锁，写操作创建新副本
4. Stack - 继承自Vector，也是线程安全的，但主要用于实现后进先出(LIFO)的堆栈

> 在并发编程中，这些线程安全的List各有优缺点：
>
> + Vector和synchronizedList使用同步锁，在高并发下性能较差
> + CopyOnWriteArrayList适合读多写少的场景，但写操作开销大
> + 对于高并发场景，可能需要考虑使用非阻塞算法的并发集合
>

### Map
1. Hashtable - 传统的线程安全Map实现，所有方法都是同步的，效率较低
2. Collections.synchronizedMap() - 将普通Map（如HashMap）包装成线程安全的版本
3. ConcurrentHashMap - 专为高并发设计的Map实现，使用分段锁技术，只锁定需要修改的部分，性能较好
4. ConcurrentSkipListMap - 基于跳表实现的有序Map，适合需要并发且保持排序的场景

> 在不同场景下的选择建议：
>
> + 一般并发场景：ConcurrentHashMap是最常用的选择，兼顾了性能和线程安全
> + 需要有序遍历：ConcurrentSkipListMap是更好的选择
> + 简单应用：Hashtable或synchronizedMap也可满足需求，但并发性能较差
>
