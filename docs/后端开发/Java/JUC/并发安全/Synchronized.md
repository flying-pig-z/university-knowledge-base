## 作用
synchronized 是Java中的一个关键字，是Java 并发编程中的同步机制之一。

当线程进入 synchronized 方法或代码块时，它会获取对应的锁。如果锁已被其他线程持有，则当前线程会被阻塞，直到锁被释放。当线程执行完同步方法或代码块时，锁会自动释放。

## 特点
1. **可重入性**：同一个线程可以多次获取同一个锁，不会导致死锁。
2. **自动释放**：无论是正常退出还是抛出异常，锁都会自动释放。
3. **不能中断**：一个线程在等待 synchronized 锁时，不能被中断。
4. **非公平性**：不保证等待线程按照请求锁的顺序获取锁。

## 原理
synchronized的底层实现涉及 JVM 的对象头、Monitor 机制以及 Java 对象模型。

#### 对象头结构
在 JVM 中，每个 Java 对象都有一个对象头，其中包含两部分信息：

+ **Mark Word**：存储对象的哈希码、GC 分代年龄、**锁状态信息**等
+ **Klass Pointer**：指向类元数据的指针

通过查看对象头的Mark Word，我们可以知道当前对象处于哪种锁状态。  

+ **无锁状态**：Mark Word存储对象的hashCode、GC分代年龄等信息
+ **偏向锁状态**：Mark Word存储持有锁的线程ID、偏向时间戳、偏向模式等
+ **轻量级锁状态**：Mark Word存储指向线程栈中Lock Record的指针
+ **重量级锁状态**：Mark Word存储指向堆中Monitor对象的指针

#### Monitor（监视器）机制
重量级锁  的实现依赖于 Monitor 机制。

每个Java对象都可以关联一个Monitor对象。Monitor提供了wait/notify机制的实现基础。

当对象的锁状态升级到重量级锁时，JVM会从ObjectMonitor池中创建/获取一个可用的Monitor对象与该Java对象关联，并将对象头中的Mark Word更新为指向该Monitor对象的指针。

Monitor 内部主要包含三个部分：

+ **Entry Set**：等待获取锁的线程集合
+ **Owner**：当前拥有锁的线程
+ **Wait Set**：调用 wait() 方法被阻塞的线程集合

#### 锁的状态与升级
JDK 1.6 引入了锁优化，使 `synchronized` 实现了从轻量级到重量级的锁状态升级过程，也称为锁膨胀。锁状态主要有四种：

【1】无锁状态

对象初始状态，未被任何线程锁定。

【2】偏向锁

+ 目的：减少同一线程获取锁的开销
+ 原理：当一个线程第一次获取锁时，会在对象头记录线程 ID，后续该线程再次获取锁，只需要比对线程 ID 即可，不需要 CAS 操作。

【3】轻量级锁

当有第二个线程尝试获取偏向锁时，偏向锁会升级为轻量级锁，通过 CAS 操作尝试在对象头中存储指向锁记录的指针。如果 CAS 成功，则线程获取锁；如果失败，则进入自旋等待。

【4】重量级锁

当自旋超过一定次数，或者多个线程竞争锁，轻量级锁会升级为重量级锁。重量级锁依赖操作系统的互斥量（Mutex）实现。线程会被挂起，进入阻塞状态，需要操作系统的调度才能再次运行。

## 锁优化技术
JDK 1.6 后，为提高 `synchronized` 性能引入了多项优化：

+ **锁消除**：JIT 编译时，对不可能存在竞争的锁进行消除
+ **锁粗化**：将多次连续加锁/解锁操作合并为一次
+ **适应性自旋**：线程等待锁时不会立即阻塞，而是执行一定次数的循环（自旋）
+ **锁升级**：从偏向锁→轻量级锁→重量级锁的渐进式升级过程

这些优化使得现代 JVM 中的 `synchronized` 性能大幅提升，在很多场景下已经接近甚至优于显式锁。
