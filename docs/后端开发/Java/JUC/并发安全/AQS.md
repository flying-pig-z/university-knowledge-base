AQS（AbstractQueuedSynchronizer）是Java并发编程中的一个核心框架。

## 什么是AQS
AQS就像是一个"排队管理系统"，专门用来管理多个线程对共享资源的访问。

## 核心组成部分
**1. 状态变量（state）**

0表示没人占用，大于0表示有人在用，通过这个数字来判断资源是否可用。

> 可能代表使用次数和可用许可数，反正就是用这个次数代表是否可用。
>

**2. 等待队列**

就像银行排队叫号系统，当资源被占用时，后来的线程就在队列里排队等待。

采用FIFO（先进先出）原则，谁先来谁先排

**3. 两种工作模式**

+ **独占模式**：同时只能一个人用（如ReentrantLock）
+ **共享模式**：可以多人同时使用（如Semaphore、CountDownLatch）

## 工作原理
```plain
1. 线程想要获取资源
2. 检查state状态是否允许获取
3. 如果可以 → 直接获取并更新state
4. 如果不可以 → 进入队列排队，然后阻塞等待
5. 当前面的线程释放资源时，唤醒队列中的下一个线程
```

## 常见的应用
+ **ReentrantLock**：可重入锁
+ **Semaphore**：信号量
+ **CountDownLatch**：倒计时门闩
+ **ReentrantReadWriteLock**：读写锁

AQS的好处是提供了一套标准的"排队等待"机制来解决并发问题。
