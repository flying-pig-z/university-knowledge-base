## Java中IO模型详解
+ UNIX下有哪五种IO模型
+ Java中常见的三种IO模型是什么?
+ 内核发起read后返回数据的三个流程是怎么样的？
+ BIO NIO IO多路复用模型 AIO分别是怎么样的？

> BIO -- 发送read命令后阻塞直到拷贝数据返回【缺点：阻塞】
>
> NIO -- 同步非阻塞，轮询调用read检测是否数据就绪，数据就绪再调用read拷贝数据返回【缺点：轮询消耗CPU资源】
>
> IO多路复用模型 -- 发送select，内核异步通知数据就绪，客户端再调用read拷贝数据返回
>
> AIO -- 完全异步，发送read先直接返回，等待应用拷贝数据后回调通知返回数据
>

NIO和IO多路复用模型并不是完全异步的，内核最后一步拷贝数据返回也是阻塞的。
