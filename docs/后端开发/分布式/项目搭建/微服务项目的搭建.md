父pom的内容 

+ 继承自SpringBoot或其他的模块 
+ 本身的坐标和打包类型 
+ modules标签聚合子模块 
+ 依赖管理和依赖（可选）

子pom的内容

+ parent标签指定继承的父模块 
+ 指定自己的artificialId和packaging 
+ 依赖和依赖管理（可选）

**重点： 父pom的modules 与 子pom的继承 缺一不可**

[1]父pom中的modules用于可以在父项目实现项目的一键构建（清理，编译，测试，打包，安装等），而不用一个个模块进行操作。

[2]子pom中的继承可以继承父模块的信息以及父模块的依赖管理和依赖。

**另外：父模块packing打包标签要为pom，而子模块则使用默认的jar就可以了。**

常见的几种打包方式

+ Jar: Jar包，java命令可以直接启动
+ Pom:用于父项目和资源项目
+ War：War包，在Tomcat等Web容器中运行
+ zip/rar：压缩文件

### 其他
（1）<relativePath>标签用于指定父POM文件的位置，以便Maven能够找到并解析父模块的配置。这个标签通常在子模块的pom.xml文件中使用。默认是<relativePath>../pom.xml</relativePath>。

在多模块项目中，父模块的packaging类型应该是pom，而不是jar。当没有使用<relativePath>指定父POM时，Maven会默认查找../pom.xml，但如果该路径下存在一个打包为jar的POM，Maven就会抛出错误。这时候需要显示指定。

（2）引入父模块无法引入子模块的类，但是可以引入子模块的依赖。
