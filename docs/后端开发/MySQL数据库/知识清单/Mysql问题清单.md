+ redo log和binlog写入时机分别是怎么样的？

> binlog是一个事务或者N个事务执行完刷盘，而redo log默认策略是事务提交刷盘，但是事务开始更新记录就在缓存中，有后台线程每隔一秒同步到page cache然后异步刷盘。
>
> 所以binlog是在事务提交完之后刷盘，而事务没有提交redo log也有可能刷盘。两者同步时间不同，就会造成两份日志不一致。
>

+ 两份日志写入时机不同造成的逻辑不一致会发生什么异常？

> 主库用redo log恢复数据，而从库用binlog恢复数据，主从之间数据不一致
>
